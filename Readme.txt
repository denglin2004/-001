/**
 ***************************************************************************************************
 * 项目名称：AI打磨机器人 — 主控制器固件
 * 项目平台：CH32V307VCT6（青稞V4F内核，RISC-V4F指令集）
 * 开发环境：MounRiver Studio（基于Eclipse）
 * 工具链  ：RISC-V GCC（MRS自带）
 * 调试器  ：WCH-Link / WCH-LinkE
 ***************************************************************************************************
 *
 * 【项目概述】
 * 本项目为AI打磨机器人的主控制器固件，负责统筹全机任务调度与模块联动控制。
 * 主控通过CAN总线与三个执行模块通信（小车底盘、升降找平、FOC打磨电机），
 * 通过RS485（DCCP协议）与大彩彩屏交互，通过UART与ESP32（云端+AI语音）通信。
 *
 * 【系统架构】
 *
 *  ┌──────────────────────────────────────────────────────┐
 *  │                   指令下发层                           │
 *  │  ┌──────────┐  ┌──────────────┐  ┌──────────────┐   │
 *  │  │  大彩彩屏  │  │ ESP32-云端   │  │ ESP32-AI语音 │   │
 *  │  │ 物理按键   │  │ 远程管控      │  │ 语音→指令    │   │
 *  │  └─────┬────┘  └──────┬───────┘  └──────┬───────┘   │
 *  │        └──────────────┼─────────────────┘            │
 *  │                       │ 统一标准指令集                  │
 *  └───────────────────────┼──────────────────────────────┘
 *                          ▼
 *  ┌──────────────────────────────────────────────────────┐
 *  │                   核心调度层                           │
 *  │              ┌──────────────────┐                    │
 *  │              │  CH32V307 主控    │                    │
 *  │              │  指令解析+任务调度 │                    │
 *  │              │  联动规则+时序管控 │                    │
 *  │              └────────┬─────────┘                    │
 *  └───────────────────────┼──────────────────────────────┘
 *                          │ CAN 总线
 *          ┌───────────────┼───────────────┐
 *          ▼               ▼               ▼
 *  ┌──────────┐   ┌──────────────┐   ┌──────────┐
 *  │ 小车底盘  │   │  升降找平模块 │   │ FOC打磨  │
 *  │  机构    │   │ 高度+平整度   │   │  电机    │
 *  └──────────┘   └──────────────┘   └──────────┘
 *                总线执行层
 *
 * 【通信接口分配】
 *
 *  端口      连接设备               波特率    协议/说明
 *  --------  -------------------  --------  --------------------------------
 *  UART1     ESP32（云端+AI语音）   115200    自定义15字节帧协议
 *  UART2     大彩彩屏（RS485）       9600    DCCP协议
 *  UART3     大彩彩屏（备用）         9600    DCCP协议
 *  UART4     预留                    —      —
 *  UART5     调试/K230视觉（预留）    —      VOFA PID调参帧
 *  CAN1      执行模块（底盘/找平/FOC） —      CAN2.0B标准帧
 *
 * 【硬件引脚定义】
 *
 *  功能          引脚              说明
 *  ------------  ----------------  ----------------------------
 *  CAN1_TX       PA12              CAN总线发送
 *  CAN1_RX       PA11              CAN总线接收
 *  USART1_TX     PA9               RS485发送（连接彩屏）
 *  USART1_RX     PA10              RS485接收（连接彩屏）
 *  OLED_SCL      PE5               OLED时钟（软件I2C）
 *  OLED_SDA      PE6               OLED数据（软件I2C）
 *  OLED_VCC      PE4               OLED电源
 *  OLED_GND      PE3               OLED地线
 *  RGB LED       bsp_rgb           状态指示灯
 *  蜂鸣器        bsp_beep          提示音
 *  ADC           bsp_adc           模拟采集
 *  SPI Flash     bsp_spi           参数存储
 *
 * 【CAN总线通信矩阵】
 *
 *  主控下发 → 执行模块：
 *  CAN-ID  目标         数据字段
 *  ------  -----------  ------------------------------------------------
 *  0x2B0   小车底盘     [0]=指令 [1]=模式 [2]=方向 [3]=速度 [4]=应答 [7]=序列号
 *  0x2B1   升降找平     [0]=功能 [1]=使能 [2]=方向 [3-4]=距离 [7]=序列号
 *  0x2B2   FOC打磨      [0]=启停 [1]=转速 [7]=序列号
 *
 *  执行模块回传 → 主控：
 *  CAN-ID  来源         数据字段
 *  ------  -----------  ------------------------------------------------
 *  0x2A0   小车底盘     应答状态
 *  0x2A1   升降找平     功能码/完成标志/当前高度/平整度
 *  0x2A2   FOC打磨      启动状态/温度/电压/电流/转速
 *
 * 【状态机设计】
 *
 *  STATE_IDLE → STATE_PREPARE → STATE_WORKING → STATE_TURNING → STATE_COMPLETE
 *
 *  主状态说明：
 *  STATE_IDLE      空闲状态，等待指令输入
 *  STATE_PREPARE   准备状态，设备归位、找平落地
 *  STATE_WORKING   工作状态，静止打磨+找平检测
 *  STATE_TURNING   转向状态，打磨头降速抬升+横向位移
 *  STATE_COMPLETE  完成状态，循环结束，设备归位
 *
 *  子状态说明：
 *  SUBSTATE_SEND_CMD      下发指令
 *  SUBSTATE_WAIT_ACK      等待CAN应答
 *  SUBSTATE_WAIT_TIME     延时等待
 *  SUBSTATE_MOVE_FORWARD  前进
 *  SUBSTATE_MOVE_BACKWARD 后退
 *  SUBSTATE_STOP          小车停止
 *  SUBSTATE_GRINDING      打磨中
 *  SUBSTATE_FOC_START     磨头启动
 *  SUBSTATE_FOC_STOP      磨头停止
 *
 * 【核心功能逻辑】
 *
 *  1. 按键控底盘移动
 *     - 四向方向键专属控制底盘，仅触发底盘动作
 *     - 按压期间持续移动，松开即刻停止
 *     - 移动过程联动管控：打磨电机自动降速+升降模块同步抬升
 *
 *  2. XY区域自动打磨
 *     - 录入X轴、Y轴区域参数作为作业指令
 *     - 主控依据参数规划打磨路径（S形路径）
 *     - 作业循环：静止打磨→找平检测→横向位移→换行复刻
 *     - 循环直至Y轴设定次数完成
 *
 *  3. 独立模块调试
 *     - 彩屏配套功能按键，可单独控制单个组件
 *     - 升降启停+定点高度设置
 *     - 底盘行驶速度调节
 *     - 打磨电机转速调节+启停
 *
 *  4. 联动保护规则（主控负责）
 *     - 底盘移动时：打磨电机自动降速+升降模块自动抬升
 *     - 打磨作业时：底盘静止+升降模块落地找平
 *     - 平整度不达标时：禁止底盘移动，等待找平完成
 *
 * 【ESP32通信协议】
 *
 *  数据帧格式（15字节固定帧）：
 *  [AA][55][A5] [设备地址] [功能码] [数据×6] [FF][FD][FF][FF]
 *
 *  设备地址：0x01=小车底盘  0x02=升降找平  0x03=FOC打磨电机
 *  功能码  ：0x01=动作控制  0x02=参数设置  0x04=一键操作
 *
 * 【大彩彩屏交互（DCCP协议）】
 *
 *  显示上传（主控→彩屏，轮询刷新）：
 *  - FOC转速/温度/电压/电流
 *  - 打磨头高度
 *  - 任务完成进度（已完成循环数/总循环数）
 *
 *  按键下发（彩屏→主控）：
 *  - 底盘方向键（按压=移动，松开=停止）
 *  - 打磨/找平控制键（启停、转速、升降）
 *  - 自动运行参数（X/Y轴打磨范围）
 *
 * 【软件分层架构】
 *
 *  APP层（应用层）
 *    ├── scheduler        任务调度器（TIM1时基驱动，10ms周期）
 *    ├── core_control     打磨作业状态机+联动规则
 *    ├── can_app          CAN协议：底盘/找平/FOC指令组包+状态解析
 *    ├── DCCP_comand      DCCP协议：彩屏指令+按钮控制+显示刷新
 *    ├── ESP32_comand     ESP32协议：云端/AI语音遥控指令
 *    └── usart_app        串口应用：端口初始化+回调注册
 *    ↓
 *  HAL层（硬件抽象层）
 *    ├── hal_can           CAN总线收发封装
 *    ├── hal_usart         串口收发封装
 *    ├── hal_rgb           RGB LED控制
 *    ├── hal_beep          蜂鸣器控制
 *    └── hal_adc           ADC采集封装
 *    ↓
 *  BSP层（板级支持包）
 *    ├── bsp_can           CAN底层驱动
 *    ├── bsp_usart         串口底层驱动
 *    ├── bsp_rgb           RGB LED底层驱动
 *    ├── bsp_beep          蜂鸣器底层驱动
 *    ├── bsp_adc           ADC底层驱动
 *    ├── bsp_spi           SPI Flash底层驱动
 *    ├── Timer             定时器驱动（TIM1/TIM8）
 *    ├── SysTick           系统滴答定时器
 *    └── Delay             延时函数
 *    ↓
 *  Peripheral层（外设库）
 *    └── ch32v30x_xxx      沁恒标准外设库（不修改）
 *
 *  其他目录：
 *    Drivers/SSD1306/      OLED驱动（软件I2C+简易UI）
 *    Middleware/            usart_protocol通用串口协议框架+check校验
 *    Debug/                 调试串口printf+VOFA PID调参帧
 *    User/                  main.c+中断服务+系统时钟
 *
 * 【定时器分配】
 *
 *  TIM1  系统时基中断（10ms周期）
 *        - PID计算
 *        - 编码器处理
 *        - RS485数据接收处理
 *        - CAN数据接收处理
 *        - USART数据接收处理
 *
 *  TIM8  系统时基中断（10ms周期）
 *        - 系统时间基准
 *        - 任务调度时钟源

 * 【开发约定】
 *
 *  1. 不修改 Peripheral/、Core/、Startup/ 目录
 *  2. 新增驱动 → BSP/（bsp_<外设>.c/h）
 *  3. 业务封装 → HAL/（hal_<外设>.c/h）
 *  4. 协议/任务 → APP/
 *  5. RISC-V编程要点：
 *     - 寄存器访问必须加 volatile
 *     - 中断服务：__attribute__((interrupt("WCH-Interrupt-fast")))
 *     - 中断控制器：PFIC（非ARM NVIC），优先级API不同
 *     - 临界区：__disable_irq() / __enable_irq()
 *     - RISC-V 4字节指令对齐
 *     - Flash 0x08000000 / RAM 0x20000000
 *     - CAN/串口数据使用 union + #pragma pack(1) 映射
