## 三层架构职责

### 1. 指令下发层
彩屏、双 ESP32 共用同一套**标准指令集**，三方均可生成操作指令并转发至主控，无缝切换、互不冲突。

| 设备 | 接口 | 功能 |
|------|------|------|
| **大彩彩屏** | RS485（DCCP 协议） | 物理按键操控：手动移动、单模块调试、XY 区域打磨参数设定、状态显示 |
| **ESP32-云端** | UART1（自定义帧） | 对接云端平台，远程下发全部控制指令，同步回传设备运行状态（电量/温度/转速/故障），异常告警 |
| **ESP32-AI语音** | （与云端共用或独立 UART） | 内置 AI 语音程序，语音识别转化为标准控制指令，支持全部设备功能 |

### 2. 核心调度层（CH32V307 主控）
- 接收所有前端转发的指令，统筹全机任务逻辑调度
- 解析指令类型，判定执行模式、动作顺序与联动规则
- 统一管控整机运行时序
- 将执行模块回传的状态数据同步反馈至彩屏和双 ESP32，形成闭环

### 3. 总线执行层（CAN 总线）
主控通过 CAN 下发控制指令，各模块独立响应总线指令并回传状态：

| 模块 | 职能 |
|------|------|
| **小车底盘机构** | 整机行走位移动作（含电机驱动 + 编码器闭环） |
| **升降找平模块** | 高度调节 + 地面平整度检测 + 调平作业 |
| **FOC 打磨电机** | 打磨电机启停 + 转速调节 + 打磨作业执行 |

## 核心功能逻辑

### 按键控底盘移动
- 四向方向键专属控制底盘，仅触发底盘动作
- **按压期间**持续移动，**松开即刻停止**
- 移动过程主控联动管控：**打磨电机自动降速 + 升降模块同步抬升**，防止磕碰磨损

### XY 区域自动打磨
1. 录入 X 轴、Y 轴区域参数作为作业指令
2. 主控依据参数规划打磨路径
3. 作业循环：
   - 小车静止 → 打磨头工作 + 升降模块落地找平
   - 模块检测地面平整度达标 → 反馈主控
   - 主控下发移动指令 → 打磨头降速 → 设备沿 X 轴行进
   - 抵达预设 X 位移 → 横向左移 → 反向复刻打磨流程
   - 循环直至 Y 轴设定次数 → 整片区域打磨完成

### 独立模块调试
彩屏配套功能按键，可单独指令管控单个组件：
- 升降启停 + 定点高度设置
- 底盘行驶速度调节
- 打磨电机转速调节 + 启停

### 联动保护规则（主控负责）
- 底盘移动时：打磨电机自动降速 + 升降模块自动抬升
- 打磨作业时：底盘静止 + 升降模块落地找平
- 平整度不达标时：禁止底盘移动，等待找平完成

## 指令闭环流程

```
彩屏/ESP32 生成指令 → 统一协议 → CH32 主控
    ↓
主控解析指令 + 联动规则 + 任务排布
    ↓
CAN 总线拆分下发 → 底盘 / 找平 / FOC 执行
    ↓
执行模块回传状态 → 主控汇总
    ↓
主控同步反馈 → 彩屏（显示）/ 云端（监测）/ 语音端（播报）
```

| 端口 | 连接设备 | 波特率 | 协议 |

| UART1 | ESP32（云端 + AI 语音） | 115200 | 15 字节帧：`AA 55 A5` 头 → 设备地址 → 功能码 → 数据 → `FF FD FF FF` 尾 |
| UART2 | 大彩彩屏（RS485） | 9600 | DCCP 协议：`EE` 头 → 控件/画面 ID → 参数 → `FF FC FF FF` 尾 |

| UART4 | 预留 | — | — |
| UART5 | 调试 / K230 视觉（预留） | — | VOFA PID 调参帧 |

## CAN 总线通信矩阵

### 主控下发 → 执行模块

| CAN-ID | 目标 | 数据字段 |
|--------|------|---------|
| `0x2B0` | 小车底盘 | [0]=指令标识 [1]=模式 [2]=方向 [3]=速度 [4]=应答 [5-6]=预留 [7]=序列号 |
| `0x2B1` | 升降找平模块 | [0]=功能 [1]=使能 [2]=方向(0停/1升/2降/3找平) [3-4]=距离参数 [5-6]=预留 [7]=序列号 |
| `0x2B2` | FOC 打磨电机 | [0]=启停使能 [1]=转速 [2-6]=预留 [7]=序列号 |

### 执行模块回传 → 主控

| CAN-ID | 来源 | 数据字段 |
|--------|------|---------|
| `0x2A0` | 小车底盘 | 应答状态 |
| `0x2A1` | 升降找平模块 | 功能码 / 完成标志 / 当前高度 / 平整度检测结果 |
| `0x2A2` | FOC 打磨电机 | 启动状态 / 温度 / 电压 / 电流 / 转速 |

## 大彩彩屏交互（DCCP 协议）

### 显示上传（主控 → 彩屏，轮询刷新）

| 显示项 | 数据来源 | 说明 |
|--------|---------|------|
| FOC 转速 | `CAN2A2_Rec.sCAN.FOC_Speed` | 打磨电机实时转速 |
| FOC 温度 | `CAN2A2_Rec.sCAN.FOC_Temp` | 打磨电机温度 |
| FOC 电压 | `CAN2A2_Rec.sCAN.FOC_VOL` | 打磨电机电压 |
| FOC 电流 | `CAN2A2_Rec.sCAN.FOC_I` | 打磨电机电流 |
| 打磨头高度 | `CAN2A1_Rec.sCAN.pzd_high` | 找平模块回传当前高度 |
| 任务完成进度 | `car_ctrl.Loop_Finished` | 已完成循环数 / 总循环数 |

### 按键下发（彩屏 → 主控）

**底盘方向键（按压=移动，松开=停止）：**
- `SET_CMD_CAR_FORWORD` (0x03) — 前进
- `SET_CMD_CAR_BACKWORD` (0x06) — 后退
- `SET_CMD_CAR_TURNLEFT` (0x04) — 左转
- `SET_CMD_CAR_TURNRIGHT` (0x05) — 右转

**打磨/找平控制键：**
- `SET_CMD_DMTSTART` (0x07) — 打磨头启停
- `SET_CMD_DMTSPEED` (0x08) — 打磨转速设定
- `SET_CMD_DMTUP` (0x0B) — 打磨头上升
- `SET_CMD_DMTDOWN` (0x0C) — 打磨头下降

**自动运行参数：**
- `0x0A` — X 轴打磨范围（Run_S_X）
- `0x0B` — Y 轴打磨范围（Run_S_Y）

### 底盘控制模式

| 模式 | ExForm_mode | 行为 |
|------|-------------|------|
| 空闲 | 0 | 无动作，等待输入 |
| 按钮手动 | 1 | 按压时长 = 运动时长，松手后按等时运动，移动中联动降速抬升 |
| 自动区域 | 2 | XY 参数驱动，自主路径规划 + 找平闭环 + 逐行打磨 |

## ESP32 通信

### 云端 ESP32
- **指令转发**：转发与彩屏同源的全部指令，远程操控所有动作与参数
- **状态监测**：采集电量、机身温度、打磨转速、故障异常等数据上传云端
- **异常告警**：识别故障状态，云端同步提示预警

### AI 语音 ESP32
- 内置 AI 语音程序，语音话术 → 标准指令集 → 主控
- 支持语音操控底盘移动、模块参数调节、区域打磨启停等全部功能

### 数据帧格式（15 字节固定帧）
```
[AA][55][A5] [设备地址] [功能码] [数据×6] [FF][FD][FF][FF]
```

| 设备地址 | 目标 |
|---------|------|
| `0x01` | 小车底盘（方向 + 速度） |
| `0x02` | 升降找平模块 |
| `0x03` | FOC 打磨电机 |

功能码：`0x01` 动作控制 / `0x02` 参数设置 / `0x04` 一键操作

## 状态机

```
STATE_IDLE → STATE_PREPARE → STATE_WORKING → STATE_TURNING → STATE_COMPLETE
```

| 主状态 | 说明 |
|--------|------|
| `STATE_IDLE` | 空闲，等待指令 |
| `STATE_PREPARE` | 准备：设备归位、找平落地 |
| `STATE_WORKING` | 打磨作业：静止打磨 + 找平检测 |
| `STATE_TURNING` | 换行转向：打磨头降速抬升 + 横向位移 |
| `STATE_COMPLETE` | 完成：循环结束，设备归位 |

子状态：`MOVE_FORWARD` / `MOVE_BACKWARD` / `STOP` / `GRINDING` / `FOC_START` / `FOC_STOP` / `WAITING`

核心参数 `Car_Ctrl_Params_t`：总步数 / 循环次数 / 前进-后退计数 / 已完成循环 / 动作-任务计时

## 软件分层

```
APP 层（应用层）
  ├── scheduler        — 任务调度器（TIM1 时基驱动）
  ├── core_control     — 打磨作业状态机 + 联动规则
  ├── can_app          — CAN 协议：底盘/找平/FOC 指令组包 + 状态解析
  ├── DCCP_comand      — DCCP 协议：彩屏指令 + 按钮控制 + 显示刷新
  ├── ESP32_comand     — ESP32 协议：云端/AI语音遥控指令
  └── usart_app        — 串口应用：端口初始化 + 回调注册
  ↓
HAL 层 — hal_can / hal_usart / hal_rgb / hal_beep
  ↓
BSP 层 — bsp_can / bsp_usart / bsp_rgb / bsp_beep / Timer / SysTick / Delay
  ↓
Peripheral 层 — 沁恒标准外设库（ch32v30x_xxx），不修改
Core / Startup — RISC-V 内核 + 启动汇编，不修改
```

其他目录：
- `Drivers/SSD1306/` — OLED 驱动（软件 I2C + 简易 UI）
- `Middleware/` — `usart_protocol` 通用串口协议框架 + `check` 校验
- `Debug/` — 调试串口 printf + VOFA PID 调参帧
- `User/` — main.c + 中断服务 + 系统时钟

## CH32 RISC-V 编程要点
- 寄存器访问必须加 `volatile`
- 中断服务：`__attribute__((interrupt("WCH-Interrupt-fast")))`
- 中断控制器：PFIC（非 ARM NVIC），优先级 API 不同
- 临界区：`__disable_irq()` / `__enable_irq()`
- RISC-V 4 字节指令对齐
- Flash `0x08000000` / RAM `0x20000000`
- CAN/串口数据使用 `union` + `#pragma pack(1)` 映射
- 串口通信统一走 `Middleware/usart_protocol` 协议框架

## 开发约定
- 不修改 `Peripheral/`、`Core/`、`Startup/`
- 新增驱动 → `BSP/`（`bsp_<外设>.c/h`）
- 业务封装 → `HAL/`（`hal_<外设>.c/h`）
- 协议/任务 → `APP/`
